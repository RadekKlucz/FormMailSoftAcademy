Formularz Kontaktowy
POST /api/contact

Limit: 5 żądań na minutę

Wymagane pola:

  • name - Imię i nazwisko (string, 2-100 znaków)
  • contact_method - Preferowana metoda kontaktu ("email" lub "phone")

Warunkowo wymagane pola:

  • email - Adres e-mail (wymagany jeśli contact_method = "email")
  • phone - Numer telefonu (wymagany jeśli contact_method = "phone")

Opcjonalne pola:

  • message - Wiadomość (string, max 2000 znaków)
  • language - Język kontaktu ("pl" lub "en", domyślnie "pl")

Uwaga: Przynajmniej jeden sposób kontaktu (email lub telefon) musi być podany.

Formularz Rezerwacyjny
POST /api/reservation

Limit: 3 żądania na minutę

Wymagane pola:

  • name - Imię i nazwisko (string, 2-100 znaków)
  • contact_method - Preferowana metoda kontaktu ("email" lub "phone")

Warunkowo wymagane pola:

  • email - Adres e-mail (wymagany jeśli contact_method = "email")
  • phone - Numer telefonu (wymagany jeśli contact_method = "phone")

Opcjonalne pola:

  • service - Typ usługi (string, max 200 znaków)
  • additional_info - Dodatkowe informacje (string, max 2000 znaków)
  • language - Język kontaktu ("pl" lub "en", domyślnie "pl")

Uwaga: Przynajmniej jeden sposób kontaktu (email lub telefon) musi być podany.

Przykładowe Żądanie

JavaScript (Fetch API)
const formData = {
    name: "Jan Kowalski",
    email: "jan@example.com",
    contact_method: "email",
    phone: "+48 123 456 789",
    message: "Witam, chciałbym uzyskać więcej informacji...",
    language: "pl"
};

fetch('/api/contact', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify(formData)
})
.then(response => response.json())
.then(data => {
    if (data.message) {
        console.log('Sukces:', data.message);
    } else {
        console.error('Błąd:', data.error);
    }
})
.catch(error => {
    console.error('Błąd sieci:', error);
});
